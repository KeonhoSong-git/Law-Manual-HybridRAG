# 변경 이력

규정 하이브리드 RAG 데모의 주요 변경·개선 기록.

## 2026-06-23 — 매뉴얼 코퍼스 확장 + 하이브리드 검색 개선

### 데이터 확장 (법령 + 행정규칙 + 매뉴얼)
- **공문체 매뉴얼 117건 추가** — 코퍼스 **517문서 · 17,340청크** (법령 127 · 행정규칙 273 · 매뉴얼 117)
- **`data_tools/fetch_manual.py` 신규** — law.go.kr 행정규칙 중 매뉴얼/안내서/가이드/운영요령류는 본문이 비고 **PDF 첨부**로만 제공 → 첨부 PDF를 받아 **pymupdf**로 텍스트 추출 후 공문체 윈도우 청킹
  - 품질 게이트: 추출 한글 글자수 임계 미만(스캔·글자깨짐 PDF)은 자동 제외
  - `--pages` 페이지네이션으로 깊은 검색 결과까지 수집
  - HWP 변환 PDF의 CID 폰트 문제로 pdfminer는 글자깨짐 → MuPDF 채택

### 임베딩 병렬화
- **`data_tools/embed_corpus.py` 다중 엔드포인트 병렬** — `EMBEDDING_API_BASE_2`(vLLM 호환 `/v1/embeddings`) 설정 시, 엔드포인트당 스레드 1개로 분산(엔드포인트 과부하 없이 throughput 약 2배)
- native(`/embeddings`)·vLLM(`/v1/embeddings`+model) 응답형식 차이를 엔드포인트별 사양으로 흡수

### 지식그래프
- **매뉴얼 `reg:Manual` 타입 부여**(`kg/build_v3.py`) — source 접두 `매뉴얼` 매핑. 이전엔 `reg:Document`로 떨어져 시드·그래프에서 누락되던 문제 해결
- **결정적 준용 추출**(`kg/classify_edges_v3.py`) — 본문 `「X」…준용` 패턴을 직접 `eli:applies` 엣지로 생성(부정문 제외, 제목 해소 실패 시 드롭). LLM 게이트가 놓치던 준용 복구 → applies 엣지 **8 → 51건**

### 데모 하이브리드 검색 (`demo/kg_api.py`)
- **타입엣지 facts 주입** — 시드 문서의 인용 관계(위임받음/준용/개정/참조)를 양방향 사실로 답변 프롬프트에 주입. 이전엔 무관한 제정일 나열
- **facts 우선순위** — 위임>준용>개정>참조 순 정렬로, 참조가 캡(20)을 채워 준용을 밀어내던 문제 해결
- **related 프루닝** — 3홉 BFS 결과를 50건 캡(`KG_MAX_RELATED`), 벡터 스코프는 시드+1홉으로 한정(818개 폭주 방지)
- **매뉴얼 시드 편입** — `_DOC_TYPES`에 `reg:Manual`·`reg:Document` 포함
